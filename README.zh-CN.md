# Course Topic Selector

[English](./README.md)

把用户提供的资料，转成**贴合市场、又不超出资料边界**的课程选题结论。

这个 skill 适合用在这些场景：
- 你手里有书稿、笔记、逐字稿、大纲、过往沉淀，但还不知道应该做什么课
- 你需要一起判断用户、竞品、定价、市场阶段和产品形态
- 你不只想知道“题目怎么起”，还想知道“这份资料到底够不够撑起一门课”

它重点解决 3 个难点：
- 这份资料最值得做成哪一个课程选题？
- 用户为什么会买这门课，而不是选工具、免费教程或服务？
- 资料是不是足够扎实，能不能撑起至少 3 节有内容的课？

## 快速调用

```text
Use $course-topic-selector to evaluate my source materials, compare market demand and competitors, and recommend the best course topic with backups, pricing, and course-readiness checks.
```

## 一眼看懂

- 输入：书稿、笔记、草稿、逐字稿、工作坊材料、作者资料库
- 输出：`1 主 2 备`、竞品分析、定价建议、MVP 范围、7 天验证计划
- 核心门槛：不仅要“适合包装”，还要“真的够做课”

## 这个 Skill 在做什么

它不是一个“起标题工具”，而是一个受约束的课程选题判断器。

它同时受两类约束：
- `资料约束`：推荐必须来自资料里真实存在的能力、观点、方法、经验、案例
- `市场约束`：推荐必须考虑真实需求、竞品、替代品、购买逻辑和替代风险

目标不是找“最大的话题”，而是找**最小但可卖的选题**：

- 作者真的讲得出来
- 用户也真的可能愿意买

## 适合什么时候用

适合：
- 课程选题
- 市场匹配度评估
- 用户定位
- 竞品和替代品分析
- 定价建议
- 决策型选题报告
- 判断资料是否足够做成一门课

不适合：
- 选题已经定了，只需要拆课
- 只需要写 lesson script
- 只需要润色文案或优化标题

## 它和普通选题方式有什么不同

这个 skill 不只会做包装，它还会明确检查：

- 选题是不是来自真实资料证据
- 用户是不是真的有这个问题
- 买家更可能买课、买工具，还是买服务
- 这份资料相比竞品有没有真正可教的内核
- 资料能不能支撑至少 3 节不注水的课程

最后这一点很关键。  
一个题目听起来成立，不代表它真的足够做成课。

## 常见输入

- 书稿
- 课程草稿
- 创始人笔记
- 工作坊逐字稿
- 研究笔记
- 作者知识库
- 经验沉淀文档

可选补充：
- 作者背景
- 结果证明
- 已知竞品
- 希望优先覆盖的市场
- 报告语言

## 常见输出

- 一个主推方向
- 两个真正不同的备选方向
- 不建议做的方向
- 竞品与替代品分析
- 推荐价格带
- 最小可上线范围
- 7 天验证计划
- 成课充分性判断
- 证据附录和追溯关系

## 输出风格

这个 skill 同时支持两种结果：

- 结构化、可追溯的决策输出
- 面向人的可读版决策报告

如果用户用某种语言发指令，最终报告会跟随该语言；调研市场也会优先覆盖该语言对应的市场。

## 核心判断逻辑

这个 skill 会按下面的顺序判断：

1. 先把资料压缩成稳定的能力核心
2. 把资料映射到真实的用户问题
3. 判断市场阶段和主流替代品
4. 确定合适的包装角度和承诺上限
5. 检查资料是否真的够做课、够支撑课时
6. 给出 `GO / HOLD / REWORK / NO-GO` 决策

## 典型使用场景

- 把一本书稿转成 `1 主 2 备` 的课程方向
- 判断一批创始人/作者笔记够不够做付费课程
- 把潜在课程拿去和工具、免费教程、训练营做比较
- 判断更适合做完整课程、工作坊、轻产品，还是应该降级

## 为什么做这个仓库

很多课程选题会失败，通常是因为两种偏差：

- 只追市场热点，最后失去作者真正的优势
- 只忠于资料本身，最后做出来的东西没人买

这个 skill 的目标，就是把这两边同时管住：  
既不虚构能力，也不忽略市场。
